Особенности обновления до версии 1.0.18.

Обязательные действия по переходу

  • Необходимо поменять обращение к регистру сведений "УсловияЗапретаПродажРМК" - регистр сведений переименован в "УдалитьУсловияЗапретаПродажРМК" в процедурах общего модуля "ОбщегоНазначенияРМКПереопределяемый":
    • ПодготовитьПреобразованиеЗапретовПродажВКонтрольПродаж(МассивОписанийКонтроля)
      • // Параметры:
        //  МассивОписанийКонтроля - Массив.

    • АктуализироватьДанныеЗапрета(ИсходныеДанные, ДанныеОбновлены)
      • // Параметры:
        //  ИсходныеДанные - Структура - содержит десериализованные данные ответа из сервера лояльности.
        //  ДанныеОбновлены - Булево - Истина, если данные успешно обновлены.

  • Необходимо поменять обращение к реквизитам справочника "Настройки рабочего места кассира" процедуре общего модуля "ОбщегоНазначенияРМКПереопределяемый":
    • СформироватьСтруктуруНастроек(СтруктураНастроек, Знач НастройкаРМК) 
      • // Параметры:
        //  СтруктураНастроек - Структура - структура, в которую нужно добавить настройки.
        Переименованы реквизиты:

  • Необходимо реализовать проверку превышения количества возвращаемых товаров перед переходом к оплате / пробитию чеков на форме рабочего места кассира. Если проверка превышения количества была реализована при записи чека, рекомендуется ее удалить оттуда, чтобы снизить время записи чека за счет исключения дублирования проверки. Проверка реализована в процедуре общего модуля "ОбщегоНазначенияРМКПереопределяемый":
    • ПроверитьВозможностьВозвратаПоЧекуККМ(Форма, КонтрольПройден, ТекстОшибки) 
    • // Параметры:
//  Форма - ФормаКлиентскогоПриложения - форма РМК,
//  КонтрольПройден - Булево - признак успешной проверки количества,
//  ТекстОшибки - Строка - текст возможной ошибки.
  • Необходимо реализовать подбор чеков коррекции аналогично подбору чеков возврата. Подбор списка чеков коррекции и возврата был разделен. Для продолжения работы с чеками коррекции необходимо реализовать соответствующую процедуру. Подбор реализован в процедуре общего модуля "ОбщегоНазначенияРМКПереопределяемый":
    • Процедура ЗаполнитьТаблицуЧековКоррекции(Форма) 
      • // Параметры:
//  Форма - ФормаКлиентскогоПриложения - форма РМК

Необязательные действия по переходу
  • Добавлена возможность определить необходимые роли для запуска РМК:
    • ДополнитьОбязательныеРолиДляЗапускаРМК(МассивРолей)
      • // Дополняет массив недостающими ролями для запуска РМК
// необходимыми в прикладном решении.
//
// Параметры:
//  МассивРолей - Массив - Массив обязательных прикладных ролей необходимых пользователю для запуска РМК.
      • Перед началом работы РМК будет проверять наличие указанных в переопределении ролей у пользователя, в случае отсутствия будет выводить соответствующее сообщение.   
  • Возможность заполнять реквизиты АдресРасчетов и МестоРасчетов при фискализации чека вынесена в отдельную процедуру:
    • ЗаполнитьПараметрыАдресаМестаРасчетов(ПараметрыОперации, ДополнительныеПараметрыДляПечати)
      • // Параметры:
        //  ПараметрыОперации - см.ОборудованиеЧекопечатающиеУстройстваКлиентСервер.ПараметрыОткрытияЗакрытияСмены().
        //  ДополнительныеПараметрыДляПечати - Структура - структура параметров печати.
      • Подразумевается 3 режима "Из ККТ", "Магазин", "Магазин+Адрес", в процедуре необходимо по переданным параметрам заполнить в ПараметрахОперации Адрес и Место расчетов.
  • Добавлена возможность переопределить возможность указания склада при продаже на форме РМК:
    • ЗаполнитьПризнакИспользованияВыборСкладаПриПродаже(ИспользоватьВыборСкладаПриПродаже, ТорговыйОбъект)
      • // Параметры:
        //  ИспользоватьВыборСкладаПриПродаже - Булево - заполняемый признак.
        //  ТорговыйОбъект - ОпределяемыйТип.ТорговыйОбъектРМК - торговый объект для которого происходит настройка.
      • Возможность добавлена на случай, если в конфигурации ведется складской учет по нескольким складам и торговый объект для входа в РМК является группой складов.
  • Добавлена новая опция ОтображатьНаименованиеДляПечати для настроек РМК, по умолчанию она не активна.
  • Добавлено переопределение используемое для получения наименования для печати номенклатуры и характеристики при формировании автоматического наименования элемента палитры товаров РМК:
    • ЗаполнитьПредставленияНоменклатурыИХарактеристики(ПредставлениеНоменклатуры, ПредставлениеХарактеристики, Номенклатура, Характеристика)
      • // Параметры:
        //  ПредставлениеНоменклатуры - Строка - Наименование для печати номенклатуры.
        //  ПредставлениеХарактеристики - Строка - Наименование для печати характеристики.
        //  Номенклатура - ОпределяемыйТип.НоменклатураРМК - Номенклатура.
        //  Характеристика - ОпределяемыйТип.ХарактеристикаРМК - Характеристика номенклатуры.

Первичное внедрение

1. Выполните объединение с файлом поставки (Конфигурация - Сравнить, объединить с конфигурацией из файла). Согласитесь с постановкой на поддержку.
2. Снимите все галочки с объединяемых объектов.
3. Выполните команду "Отметить по подсистемам файла" и отметьте только подсистему "РМК".
4. Выполнение объединения ролей прикладной конфигурации и библиотеки необходимо выполнять в режиме "Объединить с приоритетом основной конфигурации".
5. Выполните объединение конфигураций с приоритетом из файла.
6. Задайте определяемые типы, которые входят в подсистему "РМК".
7. В модуль управляемого приложения в метод "ПередНачаломРаботыСистемы" добавьте вызов процедуры "ОбщегоНазначенияРМККлиент.ПриНачалеРаботыСистемы()".
